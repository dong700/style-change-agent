# Railway 部署 - 最终修复总结

##  修复的问题

### 1. BOM 标记问题（已修复）✅

**问题**：`style_extractor.py` 文件开头有 BOM（Byte Order Mark）标记
**错误**：`SyntaxError: invalid non-printable character U+FEFF`
**影响**：Python 无法解析文件，应用启动失败，健康检查失败

**修复**：
- 使用 PowerShell 移除 BOM 标记
- 文件编码改为 UTF-8 without BOM
- 验证：本地测试通过

### 2. 配置文件路径问题（已修复）✅

**问题**：`model_config.json` 路径可能不正确
**错误**：`FileNotFoundError`

**修复**：
- 使用 `os.path.abspath(__file__)` 确保绝对路径
- 添加文件不存在时的自动创建逻辑
- 增强错误处理

### 3. 风格标签展示优化（新增功能）✅

##  常见问题

### Q1: 健康检查仍然失败

**检查**：
1. Railway Dashboard 查看实时日志
2. 查找错误信息：`Error: ...` 或 `Exception: ...`
3. 检查环境变量：`DASHSCOPE_API_KEY`

**解决**：
- 如果看到 BOM 相关错误，说明代码没有更新，需要重新推送
- 如果看到文件不存在错误，检查 `model_config.json` 是否被 Git 跟踪

### Q2: 模型切换不生效

**检查**：
1. 浏览器控制台是否有错误（F12）
2. Railway 日志是否有错误
3. `model_config.json` 是否存在

**解决**：
- 刷新页面（Ctrl+Shift+R 强制刷新）
- 检查网络连接
- 查看 Railway 日志

### Q3: 风格标签仍然是固定的

**原因**：LLM 提示词可能没有更新

**解决**：
- 确保推送了最新的 `style_extractor.py`
- Railway 重新部署完成
- 清除浏览器缓存

### Q4: 部署后功能正常但很慢

**原因**：使用的是 Qwen-Max 模型

**解决**：
- 在网页顶部切换到 Qwen-Plus 或 Qwen-Turbo
- Qwen-Turbo 速度最快（2-4 秒），成本最低
