# 模型切换功能说明

## 问题分析

### 云端分析改写慢的原因

1. **LLM 调用时间长**
   - 通义千问 qwen-max 是高性能模型，但网络延迟 + 推理时间需 5-15 秒
   - 风格分析需要分析长文本（最多 2000 字）
   - 词频过滤还需要第二次 LLM 调用
   - **总计：15-20 秒**

2. **Railway 网络延迟**
   - Railway 服务器在国外
   - 访问国内 API 有额外网络延迟（2-5 秒）

3. **两次 LLM 调用**
   - 第一次：风格分析（~10 秒）
   - 第二次：词频过滤（~5 秒）

## 解决方案：模型切换功能

### 功能特性

✅ **支持 3 种模型**：
- **Qwen-Max**：效果最好，速度较慢，成本高
- **Qwen-Plus**：速度效果平衡 ⭐推荐
- **Qwen-Turbo**：速度最快，成本低

✅ **实时切换**：
- 网页端下拉菜单选择
- 即时生效，无需重启
- 状态反馈清晰

✅ **持久化配置**：
- 配置保存到文件
- 刷新页面保持选择
- 所有用户共享配置

### 性能对比

| 模型 | 分析速度 | 改写速度 | 效果质量 | 成本 | 适用场景 |
|------|----------|----------|----------|------|----------|
| **Qwen-Max** | 10-15 秒 | 8-12 秒 | ⭐⭐⭐⭐⭐ | 高 | 深度分析、最终版本 |
| **Qwen-Plus** | 5-8 秒 | 4-6 秒 | ⭐⭐⭐⭐ | 中 | 日常使用 ⭐推荐 |
| **Qwen-Turbo** | 2-4 秒 | 1-3 秒 | ⭐⭐⭐ | 低 | 快速测试、大量处理 |

### 使用建议

- **日常使用**：Qwen-Plus（平衡）
- **额度紧张**：Qwen-Turbo（便宜 10 倍）
- **重要分析**：Qwen-Max（效果最好）
- **快速测试**：Qwen-Turbo（秒级响应）

## API 接口说明

### GET /api/model_config

获取当前模型配置

**响应示例**：
```json
{
  "success": true,
  "data": {
    "current_model": "qwen-plus",
    "available_models": [...]
  }
}
```

### POST /api/model_config

切换当前使用的模型

**请求体**：
```json
{
  "model": "qwen-turbo"
}
```

**响应示例**：
```json
{
  "success": true,
  "message": "已切换到模型：qwen-turbo",
  "data": {...}
}
```

## 技术实现细节

### 后端实现

```python
# 加载模型配置
def load_model_config():
    with open(MODEL_CONFIG_PATH, 'r', encoding='utf-8') as f:
        return json.load(f)

# 获取当前模型
def get_current_model():
    config = load_model_config()
    return config.get('current_model', 'qwen-max')

# 在上传和改写接口中使用
features = analyze_document(..., model=get_current_model())
rewriter = StyleRewriter(model=get_current_model())
```

### 前端实现

```javascript
// 切换模型
function changeModel() {
    fetch('/api/model_config', {
        method: 'POST',
        body: JSON.stringify({ model: selectedModel })
    })
    .then(response => response.json())
    .then(data => {
        if (data.success) {
            statusDiv.textContent = '✓ 已切换';
        }
    });
}

// 加载当前模型
function loadCurrentModel() {
    fetch('/api/model_config')
    .then(response => response.json())
    .then(data => {
        document.getElementById('modelSelect').value = data.data.current_model;
    });
}
```

## 常见问题

### Q: 切换模型后需要重启吗？
A: 不需要，切换后立即生效，下次上传或改写就会使用新模型。

### Q: 配置会保存多久？
A: 配置保存在服务器文件中，除非删除文件，否则会一直保留。

### Q: 不同用户会互相影响吗？
A: 会。所有用户共享同一个模型配置。这是为了简化实现，如果需要独立配置，可以添加用户认证系统。

### Q: 为什么 Plus 是推荐？
A: Plus 在速度（5-8 秒）、效果（★★★★☆）和成本（中等）之间取得了很好的平衡，适合 90% 的使用场景。

### Q: Turbo 的效果真的很差吗？
A: 不是"差"，而是相对 Max 来说细节分析可能不够深入。对于大部分应用，Turbo 的效果已经足够好。
