# LLM 提示词升级 - 多维度开放标签风格分析

## 问题诊断

### 原有问题

之前的提示词导致 LLM 输出的风格标签**非常固定**，总是出现：
- "半文半白"
- "线性编年叙事"
- "典故化修辞"
- "数据权威"

**原因**：提示词中给出了具体的标签示例，LLM 倾向于直接套用这些示例，而不是根据文本实际情况灵活分析。

### 原有提示词的问题

```python
# ❌ 旧提示词的问题
【风格标签生成指南】
请从以下维度生成风格标签（每个维度至少生成 2-3 个标签）：
1. 叙事逻辑：如"线性编年体"、"因果闭环"、"全知视角"等
2. 语言特征：如"半文半白"、"长句主导"、"庄重语体"等
3. 修辞策略：如"典故化修辞"、"辩证评价"、"多源互文"等
```

这些示例标签成为了 LLM 的"标准答案"，导致：
1. **标签固定化**：LLM 总是输出类似的标签
2. **缺乏证据**：没有强制要求提供文本依据
3. **评分模糊**：没有明确的评分标准
4. **维度封闭**：只能从预定义维度中选择，无法补充新维度

## 新提示词设计

### 核心改进

采用**多维度 + 开放标签**的设计思路：

1. **预定义核心维度**：涵盖 7 个通用维度
2. **标签完全开放**：不限制具体标签名称
3. **强制证据支持**：每个标签必须提供文本依据
4. **补充机制**：允许添加新维度
5. **自然语言总结**：提供可直接用于改写的整体描述

### 7 个核心维度

| 维度 | 说明 | 示例标签 |
|------|------|----------|
| **词汇风格** | 用词特征 | 口语化、书面化、古语词、术语密集、叠词丰富 |
| **句法风格** | 句式特征 | 长句主导、短句为主、倒装频繁、省略多 |
| **修辞风格** | 修辞手法 | 比喻、排比、反问、夸张、对偶、反讽 |
| **语气与情感** | 情感色彩 | 幽默、严肃、讽刺、感伤、激昂、平静 |
| **语体与正式度** | 文体类型 | 公文、学术、新闻、小说、日记、社交媒体 |
| **时代感与文化风格** | 时代特征 | 古风、现代、未来感、地域特色、网络流行语 |
| **叙事风格** | 叙事方式 | 第一人称、第三人称、线性、跳跃、全知、限知 |

### 新提示词模板

```python
"""你是一位专业的文本风格分析专家。请对以下文本进行深入、全面的风格分析。

【分析要求】
1. 从以下维度逐一分析文本的风格特征（每个维度可列出 1~3 个最显著的标签，每个标签给出 1-5 分并附上文本依据）：
   - 词汇风格：例如：口语化/书面化、古语词/现代词、抽象/具体、华丽/朴实、术语密度、叠词使用等。
   - 句法风格：例如：长句/短句、并列/复合句、倒装/省略、重复结构、标点特色（如破折号、省略号）等。
   - 修辞风格：例如：比喻、拟人、排比、反问、夸张、双关、对偶、反讽、借代、通感等。
   - 语气与情感：例如：幽默、严肃、讽刺、感伤、激昂、平静、亲切、冷漠、客观、主观等。
   - 语体与正式度：例如：公文/学术/新闻/广告/小说/日记/社交媒体，正式/半正式/非正式。
   - 时代感与文化风格：例如：古风、现代、未来感、地域特色（如京味、东北话）、网络流行语等。
   - 叙事风格（若适用）：例如：第一人称/第三人称、主观/客观、线性/跳跃、全知/限知视角等。

2. 如果文本还有其他上述维度未覆盖的重要风格特征（如节奏快慢、画面感强、对话性强、逻辑严密、情感充沛等），请自行在"其他显著风格"字段中补充。

3. 最后用一段话（150-250 字）总结该文本的整体风格，语言要具体、可操作，能够直接用于后续改写指导。

【重要原则】
- 标签要灵活：不要局限于示例标签，根据文本实际情况选择最合适的描述词
- 证据要充分：每个标签必须提供文本中的具体片段作为依据
- 评分要准确：1 分=不明显，3 分=中等，5 分=非常显著
- 避免套话：不要使用"半文半白"这类万能标签，要精准描述文本的真实特征

【输出格式】
请严格按照以下 JSON 结构输出，不要添加额外解释：
{
  "dimensions": {
    "词汇风格": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "句法风格": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "修辞风格": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "语气与情感": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "语体与正式度": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "时代感与文化风格": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}],
    "叙事风格": [{"label": "标签名", "score": 整数 1-5, "evidence": "依据文本片段"}]
  },
  "other_styles": [{"label": "补充标签", "score": 整数，"evidence": "依据"}],
  "summary": "整体风格描述段落"
}

【待分析文本】
{text}

请直接输出 JSON 格式结果："""
```

## 代码实现

### 修改的文件

**style_extractor.py** - `LLMStyleAnalyzer` 类

1. **更新提示词模板**（第 295-343 行）
2. **更新 `analyze` 方法**（第 387-456 行）
   - 解析新的 JSON 格式
   - 转换为前端兼容的旧格式
   - 添加维度前缀到标签名

### 格式转换逻辑

```python
# 将新格式转换为前端需要的旧格式
converted_result = {
    'overall_style_summary': result.get('summary', ''),
    'style_labels': [],
    'rewrite_suggestions': []
}

# 从各个维度提取风格标签
dimensions = result.get('dimensions', {})
for dim_name, dim_labels in dimensions.items():
    for label_item in dim_labels:
        converted_result['style_labels'].append({
            'label': f"{dim_name}: {label_item['label']}",
            'score': label_item.get('score', 3),
            'guidance': f"依据：{label_item.get('evidence', '')}"
        })
```

## 优势总结

### 1. 标签更灵活
- ✅ 不再固定输出"半文半白"等万能标签
- ✅ 根据文本实际情况选择最合适的描述
- ✅ 可以创造新的标签名称

### 2. 证据更充分
- ✅ 每个标签必须提供文本依据
- ✅ 避免 LLM 随意打分
- ✅ 便于人工校验

### 3. 维度更全面
- ✅ 7 个核心维度覆盖主要风格特征
- ✅ 允许补充新维度
- ✅ 不会遗漏重要特征

### 4. 实用性更强
- ✅ summary 可直接用于改写指导
- ✅ 评分帮助判断风格强度
- ✅ 维度分类便于理解和分析

## 后续优化建议

### 1. 温度参数调整
```python
# 当前：temperature=0（确定性最高）
# 建议：temperature=0.2-0.3（增加一点创造性）
self.llm = Tongyi(model=model, temperature=0.2)
```

### 2. 增加 Few-Shot 示例
在提示词中添加 1-2 个高质量的分析示例，帮助 LLM 理解期望的输出质量。

### 3. 标签去重
如果不同维度的标签重复，可以在后处理时合并。

### 4. 标签权重
可以根据 score 对标签排序，优先显示高分标签。

## 总结

这次升级的核心思想是：
> **维度固定，标签开放；证据充分，评分准确；总结精炼，便于使用。**

通过这种方式，LLM 能够根据文本的实际情况灵活提取风格特征，而不是机械地套用预设标签，大大提高了风格分析的准确性和实用性。
